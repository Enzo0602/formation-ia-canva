Guide 4

Connecter GitHub à Claude

Crée un token correctement, puis demande à Claude un push (juste envoyer) ou un commit (rendre public + récupérer le lien).

1
Créer un token GitHub
Settings → Developer settings → Personal access tokens → Fine-grained tokens

Sur GitHub, clique sur ta photo de profil (en haut à droite) → Settings → tout en bas de la barre latérale, Developer settings → Personal access tokens → Fine-grained tokens → Generate new token.

github.com / settings / personal-access-tokens / new

claude-formation-ia-canva
90 days ▾
All repositories
Only select repositories
📁 formation-ia-canva ▾
💡
Fine-grained token = un token limité à un seul dépôt et avec une date d'expiration. C'est le plus sûr : même partagé, il ne donne accès qu'à ce dépôt précis et devient inutilisable après l'expiration.
2
Donner les bons droits (permissions)
Repository permissions → Contents → Read and write

Toujours sur la même page, descends jusqu'à Repository permissions et règle Contents sur Read and write. C'est le seul droit nécessaire pour que Claude puisse lire et envoyer des fichiers dans ton dépôt.

github.com / settings / personal-access-tokens / new

Repository contents, commits, branches, downloads, releases

Search repositories, list collaborators (obligatoire)
Read-only ▾
Generate token

Clique sur Generate token. GitHub affiche le token une seule fois :

github_pat_11ABCDEFG0xxxxxxxxxxxxxx_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
🔒
Copie-le tout de suite et garde-le pour toi. Si tu fermes la page sans le copier, tu devras en regénérer un nouveau. Ne le mets jamais sur une page publique (Drive partagé, réseaux sociaux, etc.) — seulement dans le chat avec Claude.
3
Demander un « push » à Claude
= envoyer / mettre à jour un fichier dans le dépôt, sans forcément le rendre visible tout de suite

Un push sert à envoyer un fichier (nouveau ou modifié) dans ton dépôt GitHub. Utile quand tu veux juste sauvegarder un travail en cours, sans forcément l'ajouter à la page d'accueil.

💬 Message à envoyer à Claude
Voici mon token : github_pat_xxxxxxxxxxxx

Push ce fichier dans mon dépôt formation-ia-canva (juste l'envoyer, pas besoin de le rendre accessible publiquement pour l'instant).
✅
Claude pousse le fichier dans le dépôt. Il est sauvegardé sur GitHub, mais pas encore lié depuis ta page d'accueil ni forcément visible publiquement.
4
Demander un « commit » + récupérer le lien
= rendre le fichier accessible à tous et obtenir son URL publique

Un commit ici signifie : envoyer le fichier et le rendre consultable par n'importe qui via GitHub Pages, puis te donner le lien.

💬 Message à envoyer à Claude
Voici mon token : github_pat_xxxxxxxxxxxx

Commit ce fichier dans mon dépôt formation-ia-canva, rends-le accessible à tout le monde et donne-moi le lien public pour le voir.

Claude pousse le fichier, et te renvoie son URL publique :

Lien public du fichier
https:// enzo0602 .github.io / formation-ia-canva / mon-fichier.html

Ce lien fonctionne immédiatement pour index.html (la page d'accueil), et après 1 à 2 minutes pour les autres fichiers (le temps que GitHub Pages se mette à jour).

📤 Push
  • Envoie / met à jour un fichier
  • Rapide, pas de lien renvoyé
  • Idéal pour sauvegarder un brouillon
🌐 Commit
  • Envoie le fichier
  • Le rend accessible à tous
  • Te donne le lien public à partager
🔁
Une fois ton fichier en ligne, pense à révoquer ou regénérer ton token (étape 1) si tu n'en as plus besoin pour l'instant — c'est une bonne habitude de sécurité.
